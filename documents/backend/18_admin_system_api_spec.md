# Спецификация API: Администрирование и Безопасность (Admin System)

## Общие сведения
Модуль для управления состоянием системы, резервным копированием и глобальными настройками.
Принцип: **"Контроль над историей — контроль над будущим"**.

**Base URL:** `/admin`

---

## 1. Резервное Копирование (Backup & Restore)

### 1.1 Экспорт Состояния (Create Backup)
**Эндпоинт:** `GET /backup/export`
*Доступно только CEO.*

Возвращает полный JSON-слепок всех данных системы (БД).
**Response:** `application/json` (файл)

### 1.2 Импорт Состояния (Restore Backup)
**Эндпоинт:** `POST /backup/import`
*Доступно только CEO.*

Принимает JSON-файл, валидирует структуру и перезаписывает текущее состояние БД.
**Внимание:** Это деструктивная операция. Текущие данные будут заменены.

**Body:** (Multipart form data or JSON body with state object)

---

## 2. Управление Системой

### 2.1 Сброс к Заводским Настройкам (Hard Reset)
**Эндпоинт:** `POST /system/reset`
*Доступно только CEO.*

Очищает все данные, возвращая систему к исходному состоянию (сидеры по умолчанию).

### 2.2 Статус Системы
**Эндпоинт:** `GET /system/status`

Возвращает:
*   Версию API
*   Текущий режим (`DEVELOPMENT` / `MOBILIZATION`)
*   Статистику использования (кол-во записей в таблицах)

**Response:**
```json
{
  "version": "1.9.4",
  "mode": "MOBILIZATION",
  "stats": {
    "users": 12,
    "orders": 150,
    "warehouseItems": 450
  }
}
```
