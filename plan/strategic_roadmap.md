# Стратегическая Дорожная Карта 2.0: "Цифровой Госплан"

Этот документ сформирован на основе анализа идеологических и технических требований к системе социалистического управления производством (см. `note.md`).

**Ключевой сдвиг парадигмы:** Переход от "Учета факта" (запись того, что случилось) к "Управлению потоком" (Mobile First ввод -> Автоматический расчет -> Аналитика отклонений).

---

## Фаза 1: "Производственный Интерфейс" (Mobile First Core)
**Цель:** Упростить ввод данных "на земле". Рабочий не должен видеть сложные таблицы.

1.  **Режим "Терминал Цеха" (Kiosk Mode):**
    *   Создание упрощенного интерфейса для планшетов в цеху.
    *   **Функционал:**
        *   Авторизация по PIN-коду или NFC (быстрая смена пользователя).
        *   Кнопка **"Начать смену"** (Интеграция с модулем "Ротация" и "Посещаемость").
        *   Список назначенных на смену Производственных Заданий (ПЗ).
    *   **Ввод данных:**
        *   Кнопка **"Старт этапа"** / **"Стоп этапа"**.
        *   Кнопка **"Выпуск"**: Ввод количества -> Авто-списание сырья по нормам.
        *   Кнопка **"Инцидент"**: Быстрая фиксация брака или поломки (выбор из пресетов, фото).

2.  **Цифровой Наряд-Заказ:**
    *   Вместо абстрактных "задач" рабочий видит конкретный план на смену в натуральных показателях (штуки, кг).
    *   Визуализация прогресса: "План: 100, Сделано: 45".

---

## Фаза 2: "Материальная Диалектика" (Автоматизация расчетов)
**Цель:** Система должна сама считать экономику и трудозатраты, опираясь на первичные данные из Фазы 1.

1.  **Движок КТУ (Коэффициент Трудового Участия):**
    *   **Алгоритм:** `База (1.0)` + `Выполнение плана` + `Отсутствие брака` + `Наставничество` - `Нарушения`.
    *   Автоматический расчет ежедневного КТУ на основе закрытых нарядов.
    *   Отображение КТУ в профиле сотрудника (прозрачность начислений).

2.  **Динамическая Себестоимость:**
    *   Расчет реальной себестоимости партии в момент завершения ПЗ.
    *   Факторный анализ: `Стоимость сырья` + `Электроэнергия (по времени работы оборудования)` + `Амортизация` + `ФОТ бригады (с учетом КТУ)`.

3.  **Балансовый Метод Материалов:**
    *   Автоматическое сравнение: `Норма по Техкарте` vs `Фактическое списание`.
    *   Алерт "Перерасход": Если факт превышает норму на X%, создается Инцидент/Обсуждение для разбора причин.

## Фаза 3: "Аналитический Центр" (Обратная связь)
**Цель:** Превратить данные в управленческие решения (Демократический централизм).

1.  **Дашборд "План-Факт":**
    *   Визуализация отклонений от Техпромфинплана.
    *   Графики ритмичности производства.

2.  **Система Рационализации (Рацпредложения):**
    *   Интеграция модуля "Обсуждения" с производственными процессами.
    *   Кнопка "Предложить улучшение" прямо в карточке техпроцесса.
    *   Механизм вознаграждения: % от подтвержденной экономии идет в личный бонус или Коллективный Фонд.

3.  **Интеграция Внешнего Контура:**
    *   Портал Поставщика (упрощенный): Поставщик видит наши потребности в сырье (MRP).
    *   Портал Клиента (упрощенный): Клиент видит статус своего заказа (В производстве -> Готов).

## Техническая Реализация (Архитектурные изменения)

1.  **Refactoring `ProductionPage`:** Разделение на `ManagerView` (таблицы, планирование) и `WorkerView` (большие кнопки, степперы).
2.  **Refactoring `UserService`:** Расширение модели `User` для хранения истории КТУ и детальной статистики выработки.
3.  **Events System:** Усиление `eventManager` для триггеров (например, "ПЗ завершено" -> "Рассчитать КТУ" -> "Обновить себестоимость").
