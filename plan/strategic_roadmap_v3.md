# Стратегическая Дорожная Карта 3.1: "Кибернетический Коммунизм" (Hardened)

**Статус:** Актуальная версия (Синтез с защитными механизмами)
**Основа:** `strategic_roadmap.md` + `организация_труда_в_соц_странах.md` + *Анализ угроз отчуждения власти*.

Этот документ описывает создание системы, устойчивой к бюрократическому перерождению и произволу руководства. Код выступает гарантом справедливости.

---

## Фаза 1: "Прозрачность и Материальная Истина" (Базис)
**Цель:** Сделать ложь в системе технически невозможной или немедленно видимой.

1.  **Система "Стеклянный Завод" (Audit Log):**
    *   **Архитектура:** Создание таблицы `SystemEvents`, куда пишутся *все* изменения критических данных (Зарплата, Настройки фонда, Списание сырья).
    *   **Доступ:** Интерфейс "Журнал Событий" доступен **всем** пользователям (read-only). Рабочий видит, кто и когда изменил расценку за деталь.
    *   **Неизменяемость:** Запрет на удаление записей из лога на уровне API.

2.  **Материальные Ограничения (Physics Engine):**
    *   **Логика:** Запрет "рисовальничества".
    *   Нельзя закрыть ПЗ, если на складе нет зарезервированного сырья (блокировка кнопки "Завершить").
    *   Нельзя отгрузить Заказ, если товар не произведен (не прошел через ПЗ).
    *   *Результат:* Директор не может фиктивно списать материалы, если это не подтверждено производственным циклом.

3.  **Открытая Финансовая Сводка (Open Book):**
    *   Дашборд для всех: "Сколько заработали", "Сколько потратили на сырье", "Сколько ушло в Фонд".
    *   Скрытие *персональных* зарплат (опционально), но полная открытость *Фонда Оплаты Труда*.

---

## Фаза 2: "Распределение Власти" (Надстройка)
**Цель:** Отобрать "Красную кнопку" у одного человека.

1.  **Механизм "Двойного Ключа" (Multi-sig Approval):**
    *   Введение роли `Совет` (группа выборных пользователей).
    *   **Критические действия** (изменение % отчислений в фонд, изменение формулы КТУ, вывод средств из Фонда) требуют подтверждения (Approve) от 2-х из 3-х членов Совета.
    *   Интерфейс: CEO инициирует -> Совет получает уведомление -> Решение вступает в силу только после подписей.

2.  **Смарт-контракты Распределения:**
    *   Автоматический триггер: При закрытии месяца система *сама* рассчитывает % в Коллективный Фонд по утвержденной формуле.
    *   Никто не нажимает кнопку "Перевести". Это делает код.
    *   Ручная корректировка возможна только через публичную транзакцию "Корректировка" с обязательным комментарием, видимым всем.

## Фаза 3: "Экономика Справедливости" (Стимулы)

1.  **Алгоритм Зарплаты 2.0:**
    *   Формула: `Зарплата = (Тариф * Часы) + (Сделка) + (КТУ * Доля Прибыли)`.
    *   Тарифы жестко привязаны к грейдам (разрядам), а не устанавливаются произвольно.
    *   Визуализация: Каждый видит, из чего сложилась его сумма (как в чеке).

2.  **Рационализация как Инвестиция:**
    *   Подача рацпредложения -> Внедрение -> Замер экономии.
    *   Авто-начисление % от экономии автору. Это превращает рабочего в бенефициара эффективности.

## Техническая Реализация (Порядок работ)

### Спринт 1: "Фундамент Правды"
1.  Backend/Store: Реализация `SystemEvents` и middleware для логгирования.
2.  UI: Страница "Журнал Событий" (Global Log).
3.  UI: Блокировка действий при нарушении материального баланса (Склад <-> Производство).

### Спринт 2: "Инструменты Власти"
1.  UI: Интерфейс "Рабочий Совет" (список членов, текущие голосования).
2.  Logic: Механизм `pending_approval` для критических настроек.
3.  Logic: Авто-триггер начисления в Фонд при закрытии периода.

### Спринт 3: "Мобильный Цех"
1.  Упрощенный интерфейс для планшетов в цеху (ввод факта без лишних кликов).
2.  Визуализация КТУ и заработка в реальном времени ("Сколько я заработал сегодня").
